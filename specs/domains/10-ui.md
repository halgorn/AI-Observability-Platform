# 10 — UI Domain

Next.js 14 (App Router) + shadcn/ui + Tailwind + React Flow.

## Princípios

1. **A UI é derivada, não autoritativa.** Toda verdade vem da API.
2. **Server components por padrão**, client components só onde precisa de estado.
3. **Latência percebida < 200 ms** para interações primárias.
4. **Acessibilidade WCAG 2.1 AA.**

## §Trace — TraceView

- Árvore expansível (estilo Jaeger).
- Cada nó: `{name, duration_ms, status, agent|tool|llm_model}`.
- Click em nó = drawer lateral com `payload` + `attributes` (JSON formatado).
- Coloração: verde (OK), amarelo (warning), vermelho (error), cinza (skipped).
- Virtualização a partir de 200 nós.
- Source: `GET /v1/runs/{id}/trace`.
- Tracing alt: link "Open in Tempo" para `/tempo`/trace/{trace_id}.

## §Handoff — HandoffGraph

- React Flow, layout `dagre` (LR por padrão).
- Arestas coloridas por `success_rate`:
  - `>= 0.95` → green
  - `0.80 - 0.95` → yellow
  - `< 0.80` → red
- Click em aresta = lista de runs com falha nessa transição.
- Filtros: período (24h/7d/30d), agent subset.
- Source: `GET /v1/agents/handoffs?since=...`.

## §Replay — ReplayStepper

- Stepper horizontal: `[prev] 3/12 [next] [reset]`.
- Cada step: state diff (left = original, right = current) + eventos do step.
- Toggle: `mock_llm`, `mock_tools[]` (multi-select).
- Botão `Replay full` roda replay completo em background (worker Argo).
- Auto-advance opcional, com delay 1s.

## §Cost — CostWaterfall

- Bar chart horizontal (Recharts ou Tremor).
- 1 barra por step, ordenada por `cost_usd DESC`.
- Coloração: por `llm_model`.
- Tooltip: `tokens_in`, `tokens_out`, `cost_usd`, `prompt_version`.
- Toggle: agrupar por `agent` ou por `tool`.

## §Runs — RunsList

- Server component, paginação cursor-based.
- Colunas: `run_id`, `agent`, `status`, `started_at`, `duration_ms`, `cost_usd`, `judge_score`.
- Filtros salvos em URL (shareable).
- Default sort: `started_at DESC`.

## §Compare — CompareDiff

- Side-by-side: `run_a | run_b`.
- Header: `Δ score`, `Δ cost`, `Δ steps`, `Δ duration`.
- Diff inline em payload de eventos (igual git diff).
- Anotações do judge em tooltip.

## §Judge — JudgeScores

- Heatmap `dimension × run`.
- Score por run, sorted por timestamp.
- Click = rationale (modal).

## §SLO — SLODashboard

- Grid de cards (1 por SLO).
- Gauge: verde/amarelo/vermelho conforme `target` e janela.
- Alertas: badge "firing" se breached.

## Estado

- Server: fetch + revalidate.
- Client state: **Zustand** (leve, type-safe).
- Form state: `react-hook-form` + Zod (espelha JSON Schema).
- TanStack Query só onde precisa de cache invalidation explícita (judge polling, replay progress).

## Tema

- shadcn/ui default + overrides em `tailwind.config.ts`.
- Dark mode = default (PRD §13: "tema darktech, premium").
- Tokens: `bg-background`, `text-foreground`, `border-border`, etc.
- Cores semânticas: `--success`, `--warning`, `--error`, `--info`.

## Erros e loading

- Skeleton shadcn padrão para listas.
- Error boundary por rota (`error.tsx`).
- Empty state com CTA contextual.
- Toast (sonner) para ações destrutivas.
